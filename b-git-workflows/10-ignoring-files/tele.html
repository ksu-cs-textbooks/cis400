Ignoring Files

Git can be asked to ignore specific files by adding a pattern matching the file to a special text file named .gitignore. This file should be placed in the top level of the project directory. We typically use this to ignore temporary files, build files, and files whose contents we don’t want posted on GitHub (i.e. configuration files with passwords and secrets).

GitHub offers a collection of prepared .gitignore files for popular languages and platforms, at https://github.com/github/gitignore. You can find the one you are interested in, click the “raw” button to get the raw text, and copy it into your .gitignore file.
